# CODEX Workspace

## Purpose
This folder is the persistent operating context for future Codex sessions working on `dii.ltd`.

## Source Of Truth
- Production page: `/index.html`
- Domain config: `/CNAME`
- Social preview image: `/ILOVE.png`
- Product screenshots: `/assets/images/`
- Non-production artifact: `/tmp.html`

## Read Order Before New Work
1. `/CODEX_START_HERE.md`
2. `/CODEX/tasks/next_task.md`
3. `/CODEX/tasks/in_progress.md`
4. `/CODEX/project/current_priorities.md`
5. `/CODEX/architecture/frontend_map.md`
6. Relevant docs in `/CODEX/content/` and `/CODEX/design/`

## Folder Map
- `/CODEX/project/`: project purpose, goals, and current priorities
- `/CODEX/architecture/`: structure, file map, assets, SEO, and implementation conventions
- `/CODEX/content/`: brand voice, copy inventory, and keyword direction
- `/CODEX/design/`: visual system, UI patterns, and component inventory
- `/CODEX/tasks/`: backlog and workflow state
- `/CODEX/testing/`: QA, browser, and performance notes
- `/CODEX/logs/`: session notes and recent repo-level changes
- `/CODEX/decisions/`: durable decisions and rationale
- `/CODEX/reference/`: key paths, glossary, and audience shorthand

## Maintenance Rules
- Update docs only when changes are meaningful.
- Reference real files, sections, classes, assets, and behaviors.
- Keep task statuses current.
- If a change affects structure, copy, SEO, reusable UI, or JS behavior, update the matching docs before finishing.
